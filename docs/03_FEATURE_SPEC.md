# 03. 기능 명세 통합본

> **MVP(v1) 기준 메모:** 배경 제거는 앱 내에서 처리하며, AI agent는 사용하지 않는다. 캐릭터 탭 반응은 모든 그림에 공통으로 적용되는 기본 동작 **1종(위로 통통 튀기)** 만 제공한다. AI 기반 그림 판별 / 모션 추천 / 세분화된 터치 동작은 v2로 미룬다.

## A. 그림 가져오기 (Capture)

### A-1. 이미지 소스 선택
- **입력:** 카메라 촬영 또는 갤러리에서 기존 사진 선택
- **패키지:** `image_picker`
- **권한:** 카메라 권한 (촬영 시), 갤러리 읽기 권한
- **출력:** 원본 이미지 파일 경로

### A-2. 그림 영역 크롭
- **입력:** 원본 이미지
- **동작:**
  - 앱이 이미지 내 그림 영역을 자동 감지하여 추천 크롭 박스 표시
  - 자동 감지 방식: 이미지 가장자리 여백(흰색/단색) 영역을 분석하여 콘텐츠 바운딩 박스 추정
  - 사용자가 직접 핸들을 드래그하여 크롭 영역 조정 가능
  - **자유 직사각형** 크롭 (정사각 고정 아님)
- **패키지:** `image_cropper`
- **출력:** 크롭된 이미지 파일

### A-3. 배경 제거
- **입력:** 크롭된 이미지 (1500×1500px 초과 시 비율 유지 축소 후 처리)
- **동작:**
  1. 크롭된 이미지가 최대 해상도를 초과하면 리사이즈
  2. 이미지 모서리 영역을 샘플링해 배경색(흰색/단색) 추정
  3. 배경색과 유사한 픽셀을 threshold 기반으로 배경 후보로 분류
  4. 가장자리에서 시작하는 flood fill로 외곽 배경 확장
  5. 작은 점 노이즈 제거 후 캐릭터 외부를 투명(alpha=0) 처리
  6. 결과를 투명 배경 PNG로 인코딩
  7. 미리보기 화면에 체크무늬 배경 위에 결과 표시
- **품질 검증:**
  - 투명 영역이 전체의 95% 이상 → "그림이 잘 보이지 않아요. 다시 찍어볼까요?" 안내
  - 투명 영역이 전체의 5% 미만 → "배경을 깨끗이 지우기 어려워요. 흰 종이 위에서 다시 찍어보세요!" 안내
  - 검증 실패 시에도 `저장하기`는 허용 (사용자 판단 존중)
- **처리 중 UI:** 로딩 인디케이터 표시 (스피너 + "그림을 꾸미는 중이에요!" 텍스트)
- **사용자 액션:**
  - `저장하기` → 라이브러리에 저장
  - `저장하고 하나 더!` → 라이브러리에 저장 후 A-1(소스 선택)로 이동
  - `다시 찍기` → A-1로 돌아감
- **출력:** 투명 배경 PNG 이미지
- **비고:** v1에서는 AI / ML 모델 기반 세그멘테이션을 사용하지 않음 (01_TECH_STACK 참조)

### A-4. 라이브러리 저장
- **저장 항목:**
  - 크롭 원본 → `images/original/{id}.png`
  - 투명 배경 PNG → `images/transparent/{id}.png`
  - 썸네일 (200px 리사이즈) → `images/thumbnail/{id}.png`
  - 메타데이터 → DB (characters 테이블)
- **이름 자동 생성:** `그림 1`, `그림 2`, ... (순번 기반)

## B. 그림 라이브러리 (Library)

### B-1. 캐릭터 목록 표시
- **레이아웃:** 가로 스크롤 그리드 (3행 × N열)
- **카드 구성:** 썸네일 + 이름
- **썸네일 배경:** 체크무늬 (투명 영역 표현)
- **정렬:** 최신순 (createdAt DESC)
- **빈 상태:** 캐릭터가 0개일 때 "아직 그림이 없어요! 그림을 만들어볼까요?" 안내 + `그림 만들기` 버튼 표시

### B-2. 캐릭터 삭제
- **동작:** 길게 누르기(long press)로 삭제 모드 진입
- **삭제 시:** DB 레코드 + 파일 (original, transparent, thumbnail) 모두 삭제
- **확인:** 간단한 삭제 확인 다이얼로그

### B-3. 캐릭터 선택 (무대 연동)
- **진입 경로:** 무대 화면의 `+` 버튼 → 라이브러리 바텀시트로 열림
- **선택 시:** 해당 캐릭터를 무대에 추가하기 위한 **움직임 프리셋 선택 단계**로 이동
- **참고:** 터치 반응은 v1에서 공통 기본 동작으로 자동 적용되므로 선택 UI가 없음

## C. 무대 놀이 (Stage)

### C-1. 배경 선택
- **진입:** 홈 → `놀이 시작` → 배경 선택 화면
- **빈 라이브러리 가드:** 캐릭터가 0개일 때 `놀이 시작`을 누르면 "먼저 그림을 만들어보세요!" 토스트 표시 후 진입 차단 (캡처 화면으로 안내)
- **배경 목록 (MVP 기본 제공):**

| ID | 이름 | 설명 | 바닥선 Y 비율 |
|----|------|------|--------------|
| forest | 숲 | 초록 나무와 풀밭 | 0.80 |
| sky | 하늘 | 파란 하늘과 구름 | 0.85 |
| ocean | 바다 | 푸른 바다와 모래사장 | 0.75 |
| starry_night | 별밤 | 어두운 밤하늘과 별 | 0.85 |
| candy_land | 사탕나라 | 알록달록 과자 세상 | 0.78 |

- **선택 UI:** 가로 스크롤 카드, 탭하면 즉시 무대로 이동

### C-2. 캐릭터 추가
- **진입:** 무대 화면 하단 `+` 버튼
- **플로우:** 라이브러리 바텀시트 → 캐릭터 탭 선택 → 움직임 프리셋 선택 → `등장하기` 버튼
- **자동 적용:** 터치 반응은 `defaultBounce`로 자동 부여
- **제한:** 동시 배치 최대 10개 (성능 고려)
- **10개 도달 시:** `+` 버튼 비활성화 + 툴팁 "무대가 꽉 찼어요!" 표시
- **빈 라이브러리:** 바텀시트에 캐릭터가 없으면 "그림을 먼저 만들어볼까요?" 안내 + `그림 만들기` 버튼 (캡처 화면으로 이동)

### C-3. 움직임 프리셋

> **참고:** v1에서는 AI 추천 없이 사용자가 직접 움직임 프리셋을 선택한다.

| 프리셋 | 이름 | 애니메이션 상세 | 파라미터 |
|--------|------|----------------|----------|
| floating | 둥실둥실 | sin파 기반 상하 이동 | 진폭: 20px, 주기: 2초, easing: easeInOut |
| bouncing | 통통 점프 | 바닥에서 위로 튀기 반복 | 높이: 40px, 주기: 1.2초, easing: bounceOut |
| gliding | 씽씽 활공 | 좌우 수평 왕복 이동 | 거리: ±60px, 주기: 3초, easing: easeInOut |
| rolling | 데굴데굴 | 회전하며 수평 이동 | 회전: 360°/주기, 이동: ±80px, 주기: 2.5초 |
| spinning | 빙글빙글 | 제자리에서 회전 | 회전: 360°, 주기: 1.5초, easing: linear |

- **구현:** `AnimationController` + `Tween` 조합
- **기본 움직임은 무한 반복** (`repeat()`)
- **드래그 중에는 움직임 일시 정지**, 드래그 해제 후 재개

### C-4. 기본 터치 반응 (v1 공통)

| 프리셋 | 이름 | 애니메이션 상세 | 파라미터 |
|--------|------|----------------|----------|
| defaultBounce | 통통 점프 | 위로 튀었다 원래 위치로 복귀 | 높이: 60px, 시간: 0.5초, easing: easeOut → easeIn |

- **트리거:** 캐릭터 탭
- **대상:** 모든 캐릭터 공통
- **동작:** 기본 움직임을 일시 중단 → `defaultBounce` 재생 → 재생 완료 후 기본 움직임 재개
- **중복 탭:** 현재 터치 반응 재생 중이면 무시 (큐 없음)
- **비고:** v1에는 회전 / 반짝이기 / 카테고리별 루프 자산 없음

### C-5. 등장 애니메이션
- **동작:** scale 0 → 1.2 → 1.0 (오버슈트 바운스)
- **시간:** 0.5초
- **등장 위치:** 배경 바닥선 근처 중앙 (groundY 기준)
- **등장 후:** 자동으로 기본 움직임 시작

### C-6. 드래그 이동
- **동작:** `GestureDetector.onPanUpdate`로 캐릭터 위치 업데이트
- **범위:** 무대 영역 내로 제한 (화면 밖으로 나가지 않음)
- **드래그 시작 시:** 해당 캐릭터 zIndex를 최상위로 변경
- **드래그 중:** 기본 움직임 일시 정지
- **드래그 종료:** 기본 움직임 재개

### C-7. 다중 캐릭터 관리
- **배치:** 한 번에 하나씩 추가
- **겹치기:** 허용 (zIndex로 레이어 관리)
- **선택:** 탭한 캐릭터가 최상위로 올라옴
- **제거:** 길게 누르면 `무대에서 내리기` 옵션 표시
