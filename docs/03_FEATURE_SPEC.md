# 03. 기능 명세 통합본

## A. 그림 가져오기 (Capture)

### A-1. 이미지 소스 선택
- 경로: `/capture`
- 입력 소스:
  - 카메라
  - 갤러리
- 권한:
  - 카메라: `Permission.camera`
  - 갤러리: `Permission.photos`
- 예외 처리:
  - 권한 거부 시 피드백 배너 노출
  - 이미지 선택 취소 시 피드백 배너 노출

### A-2. 이미지 크롭
- 경로: `/capture/crop`
- 동작:
  - 진입 직후 네이티브 `image_cropper` 화면을 자동 실행
  - 크롭 취소 시 `/capture`로 복귀
  - 오류 발생 시 스낵바로 메시지 표시
  - 원본 이미지가 남아 있으면 화면 하단에서 `크롭 다시 실행` 가능
- 현재 구현 범위:
  - 자유 비율 크롭
  - 비율 프리셋 `original / 3x2 / 4x3 / 16x9`
- 현재 미구현:
  - 자동 추천 크롭 박스

### A-3. 저장 전 미리보기
- 경로: `/capture/preview`
- 현재 화면에 표시되는 것은 **배경 제거 결과가 아니라 크롭된 이미지**입니다.
- 사용자 액션:
  - `저장하기`
  - `저장하고 하나 더!`
  - `다시 찍기`
- 처리 중 UI:
  - 하단 `LinearProgressIndicator`

### A-4. 저장 시 배경 제거 및 라이브러리 등록
- 실제 배경 제거는 `저장하기` 또는 `저장하고 하나 더!`를 눌렀을 때 실행합니다.
- 처리 순서:
  1. 크롭 이미지를 원본 보관 경로로 복사
  2. 규칙 기반 배경 제거 실행
  3. 투명 PNG 기준 썸네일 생성
  4. DB에 캐릭터 메타데이터 저장
- 저장 산출물:
  - `characters/original/*.png`
  - `characters/transparent/*.png`
  - `characters/thumbnail/*.png`
  - `characters` 테이블 row
- 품질 경고:
  - 투명 비율이 `5% 미만` 또는 `95% 초과`이면 경고 문자열을 반환
  - 현재는 저장 후 완료 스낵바에 함께 표시
- 이름 정책:
  - `그림 {ISO-8601 시각}` 형식 자동 생성

## B. 그림 라이브러리 (Library)

### B-1. 기본 목록 표시
- 구현 파일: `library_screen.dart`
- 현재 라우터에는 연결되어 있지 않지만 재사용 가능한 화면으로 구현되어 있습니다.
- 정렬:
  - `createdAt DESC`
- 카드 구성:
  - 체크무늬 배경
  - 썸네일 이미지
  - 이름 라벨
- 빈 상태:
  - "아직 그림이 없어요! 그림을 만들어볼까요?"
  - `그림 만들기` 버튼

### B-2. 삭제
- 트리거:
  - 라이브러리 카드 길게 누르기
- 삭제 범위:
  - DB row 삭제
  - 원본 / 투명 / 썸네일 파일 삭제
- 확인 방식:
  - 확인 다이얼로그 후 삭제

### B-3. 무대 연동 선택
- 실제 사용 경로는 무대의 캐릭터 선택 바텀시트입니다.
- 흐름:
  1. 캐릭터 선택
  2. 모션 선택
  3. 무대에 배치

## C. 무대 놀이 (Stage)

### C-1. 배경 선택
- 경로: `/stage/background`
- 현재 UI:
  - 2열 그리드 카드
  - 카드 탭 즉시 `/stage` 이동
- 배경 목록:

| ID | 이름 | groundY |
|----|------|---------|
| `forest` | 숲 | `0.86` |
| `sky` | 푸른 하늘 | `0.84` |
| `ocean` | 바다 | `0.81` |
| `starry_night` | 별빛 밤 | `0.83` |
| `candy_land` | 캔디 랜드 | `0.88` |

### C-2. 홈에서 놀이 시작 가드
- 홈에서 `놀이 시작` 선택 시 라이브러리를 다시 로드합니다.
- 캐릭터가 없으면:
  - 스낵바 표시
  - `/capture`로 이동
- 캐릭터가 있으면:
  - `/stage/background`로 이동

### C-3. 캐릭터 추가
- 무대의 `+` floating action button으로 진입
- 바텀시트는 2단계 구조:
  1. 캐릭터 선택
  2. 모션 선택
- 최대 배치 수:
  - 10개
- 최대치 도달 시:
  - `+` 버튼 비활성화
  - 툴팁: `무대가 꽉 찼어요!`

### C-4. 모션 구조 (Object Motion / Stage Motion 분리)

무대 캐릭터의 움직임은 두 계층으로 분리합니다.

1. `objectMotion`
- 그림 자체에 적용되는 애니메이션입니다.
- 캐릭터 선택 시 고른 모션 프리셋(`floating / bouncing / gliding / rolling`)을 그대로 사용합니다.
- 스테이지 좌표 이동과 독립적으로 동작합니다.

2. `stageMotion`
- 무대 좌표에서 캐릭터의 실제 위치를 갱신하는 이동 모션입니다.
- v1에서는 `horizontalPingPong`(좌우 왕복)만 구현합니다.

### C-5. Stage Motion v1 동작 명세

- 캐릭터 생성 직후 초기 방향은 항상 `leftToRight`입니다.
- 좌우 경계에 도달하면 진행 방향을 반대로 바꿉니다.
- 경계 기준은 이미지 절반이 화면 밖으로 이동한 시점입니다.
- v1에서는 캐릭터 좌우 반전은 적용하지 않습니다.
- 결과적으로 캐릭터는 무대 안에서 좌우 왕복 이동합니다.
- 속도는 캐릭터 등장 시 `0.1 ~ 0.4` 범위에서 랜덤으로 1회 지정합니다.
- 지정된 속도는 이동 중 프레임마다 바꾸지 않고 유지합니다.
- 현재 구현은 좌우 왕복만 포함하며, 상하/대각선/경로 이동은 제외합니다.

### C-6. 등장 애니메이션
- 배치 직후 자동 재생
- scale `0.0 -> 1.2 -> 1.0`
- 총 `520ms`
- 초기 위치는 선택 배경의 `groundY`를 기준으로 중앙(`x=0.5`)에 배치

### C-7. 탭 반응
- 현재 `TouchPreset.defaultBounce`는 **위로 점프하는 이동 애니메이션이 아니라 scale bounce** 입니다.
- 값:
  - `1.0 -> 1.12 -> 1.0`
  - `300ms`
- 기본 모션은 탭 반응 동안 일시 정지 후 재개합니다.
- 탭 중복 처리:
  - 현재 반응 재생 중이면 무시

### C-8. 드래그 이동과 Stage Motion 상호작용
- `GestureDetector.onPanStart / onPanUpdate / onPanEnd` 사용
- 위치는 정규화 좌표(`0.0 ~ 1.0`)로 저장
- 드래그 시작 시:
  - 캐릭터를 맨 앞으로 이동
  - `stageMotion`을 즉시 pause
- 드래그 중:
  - 사용자 입력 위치만 반영하며 자동 이동은 없음
- 드래그 종료 시:
  - 드롭된 현재 위치를 시작점으로 유지
  - 방향은 항상 `leftToRight`로 재설정
  - 속도는 기존 값 유지(재랜덤 없음)
  - 좌우 반전은 사용하지 않으며 표시 상태 변경 없이 방향만 재설정

### C-9. 제거
- 트리거:
  - 무대 캐릭터 길게 누르기
- 현재 동작:
  - 확인 다이얼로그 없이 즉시 제거
  - 결과는 스낵바로 안내

### C-10. 무대 UI
- Android immersive fullscreen
- 상단 오버레이:
  - 뒤로가기
  - 배경 변경 버튼
  - 현재 배치 수 텍스트
  - 오류 카드
- `+` 버튼은 마지막 상호작용 이후 2초 뒤 자동 페이드아웃
- 캐릭터가 없으면 중앙 안내 문구 표시

## D. 현재 구현상 문서 메모
- `stage_painter.dart`, `background_selector.dart`, `image_processor.dart`는 placeholder 상태입니다.
- 설계 문서상 "투명 결과 미리보기"와 달리 실제 코드는 저장 시점에 배경 제거가 실행됩니다.
- 라이브러리는 현재 독립 라우트 없이 재사용 컴포넌트와 바텀시트 중심으로 사용됩니다.
