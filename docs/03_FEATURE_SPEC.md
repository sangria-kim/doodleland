# 03. 기능 명세 통합본

## A. 그림 가져오기 (Capture)

### A-1. 이미지 소스 선택
- 경로: `/capture`
- 입력 소스:
  - 카메라
  - 갤러리
- 권한:
  - 카메라: `Permission.camera`
  - 갤러리: `Permission.photos`
- 예외 처리:
  - 권한 거부 시 피드백 배너 노출
  - 이미지 선택 취소 시 피드백 배너 노출

### A-2. 이미지 크롭
- 경로: `/capture/crop`
- 동작:
  - CaptureScreen에서 자동 인식(`DrawingRegionDetector`)을 먼저 실행
  - `CropScreenArgs(sourceImagePath, detectionResult)`를 `GoRouter extra`로 전달
  - CropScreen 진입 시 `DetectionResult.boundingBox`(정규화 좌표)를 viewport 좌표로 변환해 초기 박스로 반영
  - 사용자가 크롭 영역을 조정하면 자동 인식 값보다 사용자 조정값을 우선 사용
- 현재 구현 범위:
  - 자유 비율 크롭
  - 비율 프리셋 `1:1 / 4:3 / 16:9` + 자유
  - 자동 인식 실패(`detected=false`) 시 기존 기본 초기 크롭(`viewportRect.deflate(16)`) fallback 유지

### A-2-1. DetectionResult 좌표계
- `boundingBox`는 원본 이미지 기준 정규화 좌표(`0.0~1.0`)로 정의합니다.
- 값은 `left/top/right/bottom` 비율값이며 해상도에 독립적으로 전달됩니다.
- CropScreen 연동 계층이 `imageRect` 기준으로 viewport 좌표 변환 책임을 가집니다.

### A-3. 저장 전 미리보기
- 경로: `/capture/preview`
- 현재 화면에 표시되는 것은 **배경 제거 결과가 아니라 크롭된 이미지**입니다.
- 사용자 액션:
  - `저장하기`
  - `저장하고 하나 더!`
  - `다시 찍기`
- 처리 중 UI:
  - 하단 `LinearProgressIndicator`

### A-4. 저장 시 배경 제거 및 라이브러리 등록
- 최종 투명 산출물 생성과 라이브러리 저장은 `저장하기` 또는 `저장하고 하나 더!`를 눌렀을 때 실행합니다.
- 처리 순서:
  1. 크롭 이미지를 원본 보관 경로로 복사
  2. 크롭 결과의 `nonOpaquePixelRatio(alpha < 255 비율)`를 계산
  3. `nonOpaquePixelRatio >= 5%`이면 입력 이미지를 그대로 투명 PNG로 저장하고, 미만이면 `BackgroundRemover.remove(croppedImageBytes)` 실행
  4. 투명 PNG 기준 썸네일 생성
  5. DB에 캐릭터 메타데이터 저장
- 배경 제거 마스크 구성:
  - 선화 마스크
  - 채색 마스크(Phase1 신규)
  - 경계 보존 마스크
  - 최종 결과는 세 마스크 union 후 noise component 제거
- passthrough 제한:
  - 이미 투명한 입력이라도 투명 영역이 매우 작으면 기존 remover 경로로 들어갈 수 있습니다.
- 실패 처리:
  - 자동 인식 실패 또는 배경 제거 실패 시 사용자 메시지는 `"그림을 인식하지 못했어요"`로 통일
  - 상세 실패 원인은 debug/log 데이터로만 관리
- 저장 산출물:
  - `characters/original/*.png`
  - `characters/transparent/*.png`
  - `characters/thumbnail/*.png`
  - `characters` 테이블 row
- 품질 경고:
  - remover 경로에서 계산한 `transparentAreaRatio(alpha == 0 비율)`가 `5% 미만` 또는 `95% 초과`이면 경고 문자열을 반환
  - passthrough 경로에서는 별도 품질 경고를 반환하지 않음
  - 현재는 저장 후 완료 스낵바에 함께 표시
- 이름 정책:
  - `그림 {ISO-8601 시각}` 형식 자동 생성

## B. 그림 라이브러리 (Library)

### B-1. 기본 목록 표시
- 구현 파일: `library_screen.dart`
- 경로: `/library`
- 홈의 `내 그림` 버튼에서 진입하는 전용 관리 화면으로 사용합니다.
- 정렬:
  - `createdAt DESC`
- 카드 구성:
  - 체크무늬 배경
  - 썸네일 이미지
  - 우상단 관리 메뉴 버튼
- 그리드:
  - 현재는 4열 고정 카드 목록으로 표시합니다.
- 빈 상태:
  - "아직 저장한 그림이 없어요"
  - "그림을 만들어서 나만의 그림책을 채워보세요"
  - `그림 만들기` 버튼
- 카드 탭:
  - 상세 화면으로 이동하지 않고 `준비 중인 기능이에요` 스낵바만 표시합니다.

### B-2. 삭제
- 트리거:
  - 라이브러리 카드 우상단 관리 메뉴에서 `삭제` 선택
- 삭제 범위:
  - DB row 삭제
  - 원본 / 투명 / 썸네일 파일 삭제
- 확인 방식:
  - 확인 다이얼로그 후 삭제
  - 삭제 성공 시 목록에서 즉시 제거하고 스낵바를 표시

### B-3. 무대 연동 선택
- 실제 사용 경로는 무대의 캐릭터 선택 바텀시트입니다.
- 흐름:
  1. 캐릭터 선택
  2. 모션 선택
  3. 무대에 배치
- 캐릭터 선택 카드는 `/library`와 동일한 썸네일 전용 4열 카드 스타일을 사용합니다.

## C. 무대 놀이 (Stage)

### C-1. 배경 선택
- 경로: `/stage/background`
- 현재 UI:
  - 2열 그리드 카드
  - 카드 탭 즉시 `/stage` 이동
- 배경 목록:

| ID | 이름 | groundY |
|----|------|---------|
| `forest` | 숲 | `0.86` |
| `sky` | 푸른 하늘 | `0.84` |
| `ocean` | 바다 | `0.81` |
| `starry_night` | 별빛 밤 | `0.83` |
| `candy_land` | 캔디 랜드 | `0.88` |
| `pororo_playground` | 뽀로로 놀이터 | `0.82` |

### C-2. 홈에서 놀이 시작 가드
- 홈에서 `놀이 시작` 선택 시 라이브러리를 다시 로드합니다.
- 캐릭터가 없으면:
  - 스낵바 표시
  - `/capture`로 이동
- 캐릭터가 있으면:
  - `/stage/background`로 이동

### C-3. 캐릭터 추가
- 무대의 `+` floating action button으로 진입
- 바텀시트는 2단계 구조:
  1. 캐릭터 선택
  2. 모션 선택
- 최대 배치 수:
  - 10개
- 최대치 도달 시:
  - `+` 버튼 비활성화
  - 툴팁: `무대가 꽉 찼어요!`

### C-4. 모션 구조 (Object Motion / Stage Motion 분리)

무대 캐릭터의 움직임은 두 계층으로 분리합니다.

1. `objectMotion`
- 그림 자체에 적용되는 애니메이션입니다.
- 캐릭터 선택 시 고른 모션 프리셋(`floating / bouncing / gliding / rolling`)을 그대로 사용합니다.
- 스테이지 좌표 이동과 독립적으로 동작합니다.

2. `stageMotion`
- 무대 좌표에서 캐릭터의 실제 위치를 갱신하는 이동 모션입니다.
- v1에서는 `horizontalPingPong`(좌우 왕복)만 구현합니다.

### C-5. Stage Motion v1 동작 명세

- 캐릭터 생성 직후 초기 방향은 항상 `leftToRight`입니다.
- 좌우 경계에 도달하면 진행 방향을 반대로 바꿉니다.
- 경계 기준은 이미지 절반이 화면 밖으로 이동한 시점입니다.
- v1에서는 캐릭터 좌우 반전은 적용하지 않습니다.
- 결과적으로 캐릭터는 무대 안에서 좌우 왕복 이동합니다.
- 속도는 캐릭터 등장 시 `0.1 ~ 0.4` 범위에서 랜덤으로 1회 지정합니다.
- 지정된 속도는 이동 중 프레임마다 바꾸지 않고 유지합니다.
- 현재 구현은 좌우 왕복만 포함하며, 상하/대각선/경로 이동은 제외합니다.

### C-6. 신규 배치 피드백(등장 + confetti)
- 트리거:
  - 새 캐릭터가 무대에 추가된 직후 1회
  - `instanceId` 기준으로 1회만 재생하며, 단순 리빌드/재배치 시 재생하지 않음
- 등장 애니메이션:
  - scale `0.5 -> 1.5 -> 1.0`
  - 총 `800ms`
  - scale이 주연출이며 opacity는 보조적으로 `0 -> 1` 보간(`0.0~0.62` 구간 가속)
- confetti 파티클:
  - 그림 상단보다 더 위(그림 높이의 절반 높이 위)에서 시작해 그림 하단 방향으로 낙하하는 형태
  - 소형 사각형 위주의 밝은 팔레트 파티클(`20~35`개 권장, 기본 `28`)
  - 낙하 후 수명 종료 시 opacity를 줄여 자연스럽게 제거
- 초기 위치는 선택 배경의 `groundY`를 기준으로 중앙(`x=0.5`)에 배치

### C-7. 탭 반응
- 현재 `TouchPreset.defaultBounce`는 **위로 점프하는 이동 애니메이션이 아니라 scale bounce** 입니다.
- 값:
  - `1.0 -> 1.12 -> 1.0`
  - `300ms`
- 기본 모션은 탭 반응 동안 일시 정지 후 재개합니다.
- 탭 중복 처리:
  - 현재 반응 재생 중이면 무시

### C-8. 드래그 이동과 Stage Motion 상호작용
- `GestureDetector.onPanStart / onPanUpdate / onPanEnd` 사용
- 위치는 정규화 좌표(`0.0 ~ 1.0`)로 저장
- 드래그 시작 시:
  - 캐릭터를 맨 앞으로 이동
  - `stageMotion`을 즉시 pause
- 드래그 중:
  - 사용자 입력 위치만 반영하며 자동 이동은 없음
- 드래그 종료 시:
  - 드롭된 현재 위치를 시작점으로 유지
  - 방향은 드래그 시작 전 진행 방향을 유지
  - 속도는 기존 값 유지(재랜덤 없음)
  - 좌우 반전은 사용하지 않으며 표시 상태 변경 없이 이동만 재개

### C-9. 제거
- 트리거:
  - 무대 캐릭터 길게 누르기
- 제거 상태 전이:
  1. 제거 요청 시 `removing` 상태로 전환
  2. 제거 시작 프레임의 현재 위치를 고정하고 `stageMotion`/`objectMotion`/드래그를 즉시 정지
  3. `1000ms` 퇴장 애니메이션 재생
  4. 애니메이션 완료 후 실제 리스트/위젯 트리에서 제거
- 퇴장 애니메이션 구성:
  - translateY: `0 -> +8 -> -36px` (살짝 눌린 뒤 위로 떠오름)
  - translateX: 진행 방향 기준 `±12px` 측면 이동
  - scale: `1.0 -> 1.08 -> 0.58`
  - rotation: 진행 방향 기준 `±0.16rad`
  - opacity: `1.0 -> 0.0` (`0.45~1.0` 구간 중심으로 페이드 가속)
- 예외/보호 규칙:
  - `removing` 상태에서는 탭/드래그/추가 제거 요청을 모두 무시
  - 화면 dispose 시 controller를 안전하게 해제
- 사용자 피드백:
  - 제거 시작 시 스낵바로 진행 상태를 안내

### C-10. 무대 UI
- Android immersive fullscreen
- 상단 오버레이:
  - 뒤로가기
  - 배경 변경 버튼
  - 현재 배치 수 텍스트
  - 오류 카드
- `+` 버튼은 마지막 상호작용 이후 2초 뒤 자동 페이드아웃
- 캐릭터가 없으면 중앙 안내 문구 표시

## D. 현재 구현상 문서 메모
- `stage_painter.dart`, `background_selector.dart`는 placeholder 상태입니다.
- 설계 문서상 "투명 결과 미리보기"와 달리 실제 코드는 저장 시점에 배경 제거가 실행됩니다.
- 라이브러리는 현재 독립 라우트 없이 재사용 컴포넌트와 바텀시트 중심으로 사용됩니다.
- Phase2 참고 사항(문서형/사진형 분기, 원근 보정, 모델 기반 분할)은 문서에만 유지하고 이번 구현 범위에서 제외합니다.
