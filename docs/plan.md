# 그림놀이터 구현 현황 및 잔여 과제 (2026-03-30 기준)

## 0) 현재 상태 요약
- 홈, 캡처, 크롭, 저장, 라이브러리, 무대 진입까지 MVP 기본 흐름은 구현되어 있습니다.
- 규칙 기반 배경 제거, 썸네일 생성, Drift 저장, 무대 모션 재생, 드래그/탭/제거 상호작용도 동작합니다.
- 다만 문서와 달리 저장 전 화면은 투명 결과 미리보기가 아니라 크롭 결과 확인 단계입니다.

## 1) 구현 완료 범위

- GoRouter 기반 기본 라우팅

- 캐릭터 파일 저장 경로 팩토리
- 카메라/갤러리 권한 매핑
- Android immersive fullscreen 공통 적용

### 1-2. 캡처 및 저장
- 카메라/갤러리 선택
- 네이티브 크롭 연동
- 규칙 기반 배경 제거
- 원본/투명 PNG/썸네일 저장
- 저장 후 홈 또는 재촬영 플로우

### 1-3. 라이브러리
- 캐릭터 조회
- 빈 상태 UI
- 길게 누르기 삭제

### 1-4. 무대

- 모션 5종 선택
- 등장 애니메이션
- 탭 반응, 드래그 이동, 길게 눌러 제거
- 최대 10개 배치 제한

### 1-5. 테스트 / 기본 검증
- 배경 제거 테스트
- 저장 / 삭제 유스케이스 테스트
- 라이브러리 / 홈 / 배경 선택 / 스테이지 상태 테스트
- 릴리즈 체크리스트 문서 존재

## 2) 현재 기준 잔여 과제

### 2-1. UX 정합화
- 저장 전 화면에서 실제 투명 결과를 보여주도록 플로우를 재정리할지 결정 필요
- 저장 완료 스낵바 중심 경고 전달을 화면 내 안내로 승격할지 검토 필요

### 2-2. 구조 정리
- placeholder 파일 `image_processor.dart`, `stage_painter.dart`, `background_selector.dart` 정리 필요
- stage 렌더링 구조를 계속 widget 기반으로 유지할지, 별도 렌더링 계층으로 정리할지 판단 필요

### 2-3. 기능 확장 후보
- 캐릭터 이름 수정
- 무대 씬 저장 / 불러오기
- 모션 체계 재설계
- 전체 UI 리디자인

## 3) 운영 메모
- 코드 작업은 `main`이 아닌 `feature/...` 브랜치에서 진행합니다.
- 문서 전용 수정은 `main`에서 허용되지만 커밋 메시지 컨벤션은 동일하게 유지합니다.
- `main`에 새 커밋이 추가되면 `docs/PROGRESS.md`를 함께 갱신합니다.

## 4) Phase1 구현 계획: 그림 자동 인식 + 배경 제거 개선 (2026-04-05)

### 4-1. 이번 구현 범위 (Phase1만)
- 스케치북/A4 종이에 그린 그림을 대상으로 자동 인식과 배경 제거 품질을 개선합니다.
- Capture 단계에서 자동 인식을 실행하고, 결과를 Crop 단계 초기 박스로 전달합니다.
- 배경 제거는 인터페이스 중심 구조로 재정리하고, 선화/채색/경계 보존 마스크 union 구조를 적용합니다.
- 자동 인식 실패 시에는 기존 기본 크롭 상태를 그대로 fallback으로 사용합니다.
- `confidence`는 저장하되 이번 버전 UI 분기에는 사용하지 않습니다.

### 4-2. 제외 범위
- 원근 보정(Homography/4점 변환), OpenCV 컨투어 기반 문서 검출, 모델 기반 범용 분할은 제외합니다.
- 일반 사진(사람/사물/동물) 대상 범용 foreground segmentation은 제외합니다.
- `confidence` 기반 fallback/UI 분기 로직은 이번 작업에 포함하지 않습니다.

### 4-3. 아키텍처 원칙
- 자동 인식(`DrawingRegionDetector`)과 배경 제거(`BackgroundRemover`)는 별도 모듈로 분리합니다.
- 두 모듈은 입력/출력 인터페이스로만 연결하고, UI는 내부 알고리즘 세부사항을 모르게 유지합니다.
- `DetectionResult.boundingBox`는 원본 이미지 기준 정규화 좌표(`0.0~1.0`)로 고정합니다.
- Capture -> Crop 전달은 공유 상태가 아닌 `GoRouter extra`의 args 객체로 단방향 전달합니다.

### 4-4. 구현 우선순위
1. `DrawingRegionDetector` 인터페이스와 기본 구현체를 추가합니다.
2. CaptureScreen에서 자동 인식을 수행하고 `CropScreenArgs`로 전달합니다.
3. CropScreen에서 정규화 bbox를 viewport 좌표로 변환해 초기 크롭 박스를 설정합니다.
4. `BackgroundRemover`를 인터페이스 기반으로 추상화하고 크롭 결과 바이트 입력으로 단순화합니다.
5. 채색 마스크를 별도 패스로 추가하고 선화/채색/경계 보존 마스크 union 구조를 적용합니다.
6. 실패 노출 문구를 `"그림을 인식하지 못했어요"`로 단일화하고 debug/release 출력을 분리합니다.

### 4-5. Phase2 참고(이번 구현 제외)
- 향후 입력 이미지를 문서형(종이 위 그림)과 사진형으로 분기하는 구조를 검토합니다.
- 문서형은 Phase1 파이프라인을 유지하고, 사진형은 별도 segmentation 파이프라인으로 확장합니다.
