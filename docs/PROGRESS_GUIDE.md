> 이 문서는 모든 AI Agent가 
> 그림놀이터 앱 개발 진행 상황을 추적하기 위해 참고하는 지침입니다.

---

## 📋 PROGRESS.md 구조

```
# 그림놀이터 앱 개발 진행 현황

## 📊 전체 진행률
[카테고리별 진행률 테이블]

## 🚧 현재 상태
- 진행 중인 작업
- 해결 필요한 블로킹 이슈
- 다음 예정 작업

## 📝 커밋 로그
### [날짜 YYYY-MM-DD]
#### [커밋 메시지] (`commit: [짧은 커밋ID]`)
- 구현 내용 1
- 구현 내용 2
- 구현 내용 3
```

## 🔄 업데이트 방법

### **1단계: 작업 완료**
```bash
# feature branch 또는 별도 작업 환경에서 개발 완료
git checkout main
git pull  # 최신 상태 반영
```

### **2단계: main 브랜치에 커밋 추가**

다음 중 하나의 방식으로 main에 커밋 추가:

**방식 A: squash merge (권장)**
```bash
git merge --squash feature/[브랜치명]
git commit -m "feat: 기능 설명"
```

**방식 B: 직접 커밋**
```bash
# 작업 완료 후 직접 커밋
git add .
git commit -m "feat: 기능 설명"
```

**방식 C: 다른 도구/Agent 사용**
- GitHub UI, GitLab UI 등을 통한 merge
- 다른 CI/CD 파이프라인
- 기타 git 관리 도구

**중요**: 어떤 방식이든 **main 브랜치에 새로운 커밋이 추가되면** PROGRESS.md를 업데이트해야 합니다.

### **3단계: PROGRESS.md 업데이트**

#### **A. 📊 전체 진행률 업데이트**
- 해당 카테고리의 진행률(%) 수정
- 상태 업데이트: 🔄 진행 중 → ✅ 완료 (또는 다른 상태)

```markdown
| **UI 컴포넌트** | 50% | 🔄 |  # 0% → 50%로 변경
```

#### **B. 🚧 현재 상태 업데이트**
- **진행 중인 작업**: 현재 진행 중인 작업 또는 "없음"
- **해결 필요한 블로킹 이슈**: 막혀있는 이슈 또는 "없음"
- **다음 예정 작업**: 다음에 할 작업 또는 "없음"

```markdown
## 🚧 현재 상태

**진행 중인 작업**
- [현재 진행 중인 작업 설명 또는 "없음"]

**해결 필요한 블로킹 이슈**
- [블로킹 이슈 또는 "없음"]

**다음 예정 작업**
- [다음에 할 작업 또는 "없음"]
```

#### **C. 📝 커밋 로그에 추가**

**위치**: 최신 날짜 섹션의 맨 위에 추가 (최신순)

**날짜 섹션이 없으면 새로 생성**:
```markdown
### 2026-03-28
```

**형식**:
```markdown
#### feat: UI 기본 구성 (캔버스, 도구 바, 네비게이션) (`commit: a1b2c3d`)
- Canvas CustomPaint로 구현
- 도구 바 (색상, 브러시, 지우개) UI 완성
- Riverpod StateNotifier로 상태 관리
```

**규칙**:
- 커밋 제목 형식: `타입: 간단한 설명`
  - 타입: `feat`, `fix`, `docs`, `refactor`, `test` 등
- 커밋 제목 옆에 `commit: [짧은 커밋ID]`를 반드시 함께 기록한다.
  - 예: ``(`commit: 4f931b2`)``
- 설명: 2-3줄의 간단한 구현 내용 (bullet list)
- 같은 날짜의 여러 커밋: 같은 날짜 섹션 아래 계속 추가
- **최신 커밋이 맨 위에 오도록 정렬** (위쪽이 더 최신)

## ✅ 체크리스트

각 업데이트 시 확인:

- [ ] main 브랜치에 새로운 커밋이 추가되었는가?
- [ ] 커밋 메시지가 명확한가? (타입: 기능 설명 형식)
- [ ] 커밋 로그 항목에 `commit id`를 함께 기록했는가?
- [ ] 커밋 설명이 2-3줄로 작성되었는가?
- [ ] 📊 전체 진행률 테이블을 업데이트했는가?
- [ ] 🚧 현재 상태의 3가지 항목을 업데이트했는가?
- [ ] 📝 커밋 로그에 새 커밋을 추가했는가?
- [ ] 커밋 로그의 날짜 섹션이 최신순인가? (위쪽이 더 최신)
- [ ] 같은 날짜의 커밋들이 최신순인가? (위쪽이 더 최신)

## 💡 팁

**1. 진행률 업데이트 기준**
```
0%   → 아직 시작 안 함
25%  → 초기 구조 및 기본 틀 완료
50%  → 핵심 기능 구현 중
75%  → 대부분 완료, 마무리/테스트 중
100% → 완전히 완료
```

**2. 블로킹 이슈 기재 예시**
- "ML Kit Subject Segmentation 성능 문제"
- "Riverpod FamilyStateNotifier 복잡도 증가"
- "이미지 처리 라이브러리 선택 필요"

**3. 커밋 타입 가이드**
```
feat:     새 기능 추가
fix:      버그 수정
refactor: 코드 구조 개선 (기능 변경 없음)
docs:     문서 작성/수정
test:     테스트 코드 추가
perf:     성능 최적화
style:    포맷팅, 세미콜론 등 (의미 있는 코드 변경 없음)
```

**4. 커밋 메시지 예시**
```
feat: 사용자 인증 모듈 구현
- Firebase Auth 연동
- JWT 토큰 관리
- 로그인/로그아웃 UI 완성

fix: 캔버스 렌더링 버그 수정
- 고해상도 이미지에서 스트로크 끊김 현상 해결
- 메모리 누수 제거
- 성능 테스트 통과

docs: 아키텍처 문서 작성
- Clean Architecture 계층 설명
- 의존성 주입 패턴 설명
- 상태 관리 흐름 다이어그램 추가
```

## 🎯 핵심 원칙

1. **main 브랜치의 커밋만 기록** (feature branch 작업 내역은 기록 X)
2. **최신순으로 정렬** (위쪽이 더 최신)
3. **커밋 메시지 + commit id 동시 기록**
4. **매 커밋마다 🚧 현재 상태 함께 갱신**
5. **어떤 도구/Agent에서든 동일한 형식 유지**
