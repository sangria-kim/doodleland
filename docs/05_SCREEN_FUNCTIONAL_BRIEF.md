# 05. 화면 기능 설명서

이 문서는 생성형 AI에게 화면 디자인을 요청할 때 사용할 수 있는 **기능 중심 설명서**입니다.
색상, 분위기, 폰트, 레이아웃 스타일 같은 디자인 지시는 제외하고, 각 화면이 반드시 수행해야 하는 역할과 상태만 정리합니다.

## 공통 조건
- 앱은 가로(landscape) 고정입니다.
- 주요 대상은 아이와 보호자이지만, 이 문서에서는 사용성 설명보다 기능 요구사항만 다룹니다.
- 현재 라우트는 아래 6개입니다.

| 화면 | 경로 | 비고 |
|------|------|------|
| 홈 | `/` | 앱 진입 화면 |
| 그림 가져오기 | `/capture` | 카메라/갤러리 선택 |
| 이미지 크롭 | `/capture/crop` | 네이티브 크롭 진입용 화면 |
| 미리보기 | `/capture/preview` | 저장 전 크롭 결과 확인 |
| 배경 고르기 | `/stage/background` | 무대 배경 선택 |
| 무대 | `/stage` | 캐릭터 배치 및 상호작용 |

추가로, 무대에서 여는 **캐릭터 선택 바텀시트**가 있습니다. 이 바텀시트는 별도 화면처럼 취급해 설명합니다.

## 1. 홈 화면

### 목적
- 앱의 첫 진입점입니다.
- 사용자가 그림을 새로 만들지, 기존 그림으로 무대 놀이를 시작할지 선택합니다.

### 진입 조건
- 앱 실행 시 기본 진입 화면입니다.
- 저장 완료 후 `저장하기`를 선택하면 다시 이 화면으로 돌아옵니다.

### 반드시 포함되어야 하는 기능
- 앱 제목 또는 앱 식별 텍스트
- `그림 만들기` 액션
- `놀이 시작` 액션

### 사용자 액션
- `그림 만들기` 선택 시 `/capture`로 이동
- `놀이 시작` 선택 시 라이브러리 데이터를 다시 확인

### 상태 / 예외 처리
- 라이브러리에 그림이 하나도 없으면:
  - 스낵바 메시지 표시
  - `/capture`로 이동
- 라이브러리에 그림이 있으면:
  - `/stage/background`로 이동

## 2. 그림 가져오기 화면

### 목적
- 그림의 입력 소스를 선택합니다.

### 진입 조건
- 홈의 `그림 만들기`
- 홈에서 빈 라이브러리 상태로 `놀이 시작`
- 미리보기에서 `저장하고 하나 더!`
- 미리보기에서 `다시 찍기`

### 반드시 포함되어야 하는 기능
- 화면 제목
- `카메라로 찍기` 버튼
- `갤러리에서 선택` 버튼
- 처리 중 진행 상태 표시 영역
- 권한/취소/오류 메시지 표시 영역

### 사용자 액션
- 카메라 선택
- 갤러리 선택

### 상태 / 예외 처리
- 카메라 권한 거부 시 메시지 표시
- 갤러리 권한 거부 시 메시지 표시
- 이미지 선택 취소 시 메시지 표시
- 이미지 선택 성공 시 `/capture/crop`로 이동하며 원본 경로 전달
- 처리 중에는 중복 입력을 막아야 함

## 3. 이미지 크롭 화면

### 목적
- 사용자가 선택한 원본 이미지에 대해 크롭 작업을 수행합니다.
- 실제로는 네이티브 cropper를 호출하는 중간 화면 역할을 합니다.

### 진입 조건
- `/capture`에서 이미지 선택 성공 시 진입

### 반드시 포함되어야 하는 기능
- AppBar 또는 화면 제목
- 현재 선택된 원본 이미지의 확인 영역
- `크롭 다시 실행` 버튼
- 처리 중 진행 상태 표시 영역

### 사용자 액션
- 네이티브 크롭 UI에서 자르기
- 네이티브 크롭 UI에서 취소
- 필요 시 `크롭 다시 실행`

### 상태 / 예외 처리
- 진입 직후 자동으로 네이티브 cropper 호출
- 크롭 성공 시 `/capture/preview`로 이동하며 크롭 결과 경로 전달
- 크롭 취소 시 `/capture`로 복귀
- 원본 이미지가 없으면 오류 메시지 표시
- 크롭 오류 시 스낵바 메시지 표시
- 처리 중에는 `크롭 다시 실행` 비활성화

## 4. 미리보기 화면

### 목적
- 저장 전에 현재 크롭 결과를 확인합니다.
- 저장 액션을 기준으로 실제 배경 제거와 라이브러리 저장을 수행합니다.

### 진입 조건
- 크롭 성공 후 `/capture/preview`로 이동

### 반드시 포함되어야 하는 기능
- AppBar 또는 화면 제목
- 현재 크롭 결과 이미지 표시 영역
- `저장하기`
- `저장하고 하나 더!`
- `다시 찍기`
- 저장 중 진행 상태 표시 영역

### 중요한 기능 메모
- 이 화면은 **배경 제거 결과 미리보기 화면이 아닙니다.**
- 실제 배경 제거는 `저장하기` 또는 `저장하고 하나 더!`를 눌렀을 때 실행됩니다.

### 사용자 액션
- `저장하기`
- `저장하고 하나 더!`
- `다시 찍기`

### 상태 / 예외 처리
- 저장 중에는 중복 액션 방지
- 저장 성공 시:
  - `저장하기` -> `/`
  - `저장하고 하나 더!` -> `/capture`
- 저장 실패 시 스낵바 메시지 표시
- 배경 제거 품질 경고가 있으면 저장 완료 스낵바에 함께 표시

## 5. 배경 고르기 화면

### 목적
- 무대에 사용할 배경을 선택합니다.

### 진입 조건
- 홈에서 `놀이 시작` 성공 시 진입
- 무대에서 배경 변경 버튼 선택 시 진입

### 반드시 포함되어야 하는 기능
- 화면 제목
- 배경 목록
- 각 배경의 이미지와 이름

### 배경 목록
- 숲
- 푸른 하늘
- 바다
- 별빛 밤
- 캔디 랜드

### 사용자 액션
- 배경 카드 선택

### 상태 / 예외 처리
- 배경 선택 시 즉시 선택 상태를 저장
- 선택 후 `/stage`로 이동
- 무대에서 다시 들어온 경우에도 같은 경로를 사용

## 6. 무대 화면

### 목적
- 선택한 배경 위에 저장된 캐릭터를 배치하고 상호작용합니다.

### 진입 조건
- 배경 선택 완료 후 진입

### 반드시 포함되어야 하는 기능
- 선택된 배경 전체 표시
- 뒤로가기 액션
- 배경 변경 액션
- 현재 배치 수 표시 (`현재 수 / 10`)
- 오류 메시지 표시 영역
- 캐릭터 추가 버튼
- 빈 무대 안내 상태

### 캐릭터 관련 기능
- 무대에 캐릭터 여러 개 배치 가능
- 같은 캐릭터를 여러 번 배치 가능
- 최대 10개까지 배치 가능
- 캐릭터는 각자 선택한 모션을 자동 재생
- 배치 직후 등장 애니메이션 재생

### 사용자 액션
- `+` 버튼으로 캐릭터 선택 바텀시트 열기
- 뒤로가기 -> `/`
- 배경 변경 -> `/stage/background`
- 캐릭터 탭
- 캐릭터 드래그
- 캐릭터 길게 누르기

### 상태 / 예외 처리
- 배치된 캐릭터가 없으면 빈 상태 안내 표시
- 최대 10개 도달 시:
  - `+` 버튼 비활성화
  - 꽉 찼다는 안내 필요
- 오류 메시지가 있으면 화면 상단 영역에 표시
- 마지막 상호작용 이후 `+` 버튼은 숨겨졌다가, 다시 터치하면 보일 수 있어야 함

## 7. 캐릭터 선택 바텀시트

### 목적
- 무대에 올릴 캐릭터와 모션을 선택합니다.

### 진입 조건
- 무대 화면의 `+` 버튼

### 전체 흐름
1. 캐릭터 선택
2. 모션 선택
3. 확인 후 무대에 배치

### 1단계: 캐릭터 선택

#### 반드시 포함되어야 하는 기능
- 저장된 캐릭터 목록
- 각 캐릭터의 썸네일
- 빈 상태 안내
- `그림 만들기` 이동 액션

#### 상태 / 예외 처리
- 라이브러리 로딩 중 상태 필요
- 캐릭터가 없으면:
  - 빈 상태 메시지 표시
  - `그림 만들기` 버튼 제공
  - 버튼 선택 시 바텀시트를 닫고 `/capture`로 이동

### 2단계: 모션 선택

#### 반드시 포함되어야 하는 기능
- 선택한 캐릭터 미리보기
- 현재 선택된 모션 이름
- 모션 설명
- 모션 선택 목록
- 뒤로가기
- 배치 확인 액션

#### 모션 목록
- 둥실둥실
- 통통 점프
- 씽씽 활공
- 데굴데굴

#### 상태 / 예외 처리
- 기본 선택 모션은 `둥실둥실`
- 뒤로가기를 누르면 캐릭터 선택 단계로 복귀
- 확인 시 선택한 캐릭터와 모션 정보를 무대 화면으로 반환

## 8. 무대 캐릭터 상호작용 규칙

이 항목은 무대 화면 디자인 요청 시 함께 전달할 기능 규칙입니다.

### 탭
- 해당 캐릭터를 맨 앞으로 올림
- 기본 모션을 잠시 멈춤
- scale bounce 반응 재생
- 반응 종료 후 기본 모션 재개
- 반응 재생 중 추가 탭은 무시

### 드래그
- 드래그 시작 시 캐릭터를 맨 앞으로 올림
- 드래그 중 기본 모션 정지
- 드래그 종료 후 기본 모션 재개
- 위치는 무대 내부 기준 좌표로 관리

### 길게 누르기
- 해당 캐릭터를 즉시 무대에서 제거
- 제거 결과는 스낵바 메시지로 안내


## 생성형 AI 전달용 전체 텍스트

```text
가로 모드 전용 Flutter 앱의 전체 화면 디자인을 제안해주세요. 디자인 스타일은 새롭게 구성해도 되지만, 아래 기능 구조와 화면 전환은 유지해야 합니다.

앱은 총 6개의 주요 화면과 1개의 바텀시트로 구성됩니다.
주요 화면은 홈, 그림 가져오기, 이미지 크롭, 미리보기, 배경 고르기, 무대입니다.
추가로 무대에서 열리는 캐릭터 선택 바텀시트가 있습니다.

홈 화면은 앱의 시작 화면입니다.
사용자는 여기서 그림 만들기 또는 놀이 시작 중 하나를 선택합니다.
그림 만들기를 누르면 그림 가져오기 화면으로 이동합니다.
놀이 시작을 누르면 저장된 그림이 있는지 확인한 뒤, 그림이 없으면 안내 메시지를 보여주고 그림 가져오기 화면으로 이동하며, 그림이 있으면 배경 고르기 화면으로 이동합니다.

그림 가져오기 화면에서는 입력 소스를 선택해야 합니다.
사용자는 카메라로 찍기 또는 갤러리에서 선택 중 하나를 고를 수 있어야 합니다.
권한이 없거나 이미지 선택을 취소한 경우 메시지를 보여줄 수 있어야 합니다.
이미지를 정상적으로 선택하면 이미지 크롭 화면으로 이동합니다.

이미지 크롭 화면은 실제로 네이티브 크롭 UI를 호출하는 중간 화면입니다.
진입 직후 자동으로 크롭 UI가 실행되어야 합니다.
크롭에 성공하면 미리보기 화면으로 이동합니다.
크롭을 취소하면 그림 가져오기 화면으로 돌아갑니다.
오류가 발생하거나 원본 이미지가 없을 때를 위한 안내가 필요합니다.
또한 사용자가 크롭 다시 실행 버튼으로 동일 이미지를 다시 자를 수 있어야 합니다.

미리보기 화면은 저장 전 확인 화면입니다.
중요한 점은 이 화면에서 보여주는 것이 배경 제거 결과가 아니라 크롭된 이미지라는 점입니다.
사용자는 저장하기, 저장하고 하나 더, 다시 찍기 3가지 액션을 사용할 수 있어야 합니다.
실제 배경 제거와 라이브러리 저장은 저장 버튼을 눌렀을 때 실행됩니다.
저장 성공 시 저장하기는 홈으로 이동하고, 저장하고 하나 더는 그림 가져오기 화면으로 이동합니다.
저장 실패나 품질 경고가 있을 경우 메시지를 표시할 수 있어야 합니다.

배경 고르기 화면에서는 무대에 사용할 배경을 선택합니다.
현재 배경 종류는 숲, 푸른 하늘, 바다, 별빛 밤, 캔디 랜드 5개입니다.
각 배경은 이미지와 이름을 함께 보여줘야 하며, 하나를 선택하면 즉시 무대 화면으로 이동합니다.
이 화면은 홈에서 놀이 시작 후 진입할 수도 있고, 무대에서 배경 변경을 눌렀을 때 다시 진입할 수도 있습니다.

무대 화면은 핵심 놀이 화면입니다.
선택된 배경이 전체에 표시되어야 하고, 저장된 캐릭터들을 무대 위에 여러 개 배치할 수 있어야 합니다.
상단에는 뒤로가기, 배경 변경, 현재 배치된 캐릭터 수, 오류 메시지 영역이 필요합니다.
우하단에는 캐릭터 추가 버튼이 필요합니다.
무대가 비어 있을 때는 안내 상태가 보여야 합니다.

무대에서는 같은 캐릭터도 여러 번 배치할 수 있습니다.
배치 가능한 최대 수는 10개입니다.
10개에 도달하면 추가 버튼은 비활성화되어야 하고, 더 이상 배치할 수 없다는 안내가 필요합니다.
각 캐릭터는 배치 직후 등장 애니메이션이 재생되고, 이후에는 선택한 기본 모션을 계속 반복합니다.

캐릭터 상호작용 규칙도 포함해야 합니다.

탭 반응 재생 중 중복 탭은 무시합니다.
캐릭터는 드래그로 위치를 이동할 수 있어야 하며, 드래그 중에는 기본 모션이 멈추고 드래그가 끝나면 다시 재생되어야 합니다.
캐릭터를 길게 누르면 무대에서 즉시 제거되어야 합니다.

1단계에서는 저장된 캐릭터 목록을 보여주고, 사용자가 하나를 고르면 2단계로 넘어갑니다.
2단계에서는 선택한 캐릭터의 미리보기와 함께 모션을 고를 수 있어야 합니다.
모션 종류는 둥실둥실, 통통 점프, 씽씽 활공, 데굴데굴 4가지입니다.
기본 선택 모션은 둥실둥실입니다.
사용자는 뒤로가기로 다시 캐릭터 선택 단계로 돌아갈 수 있어야 하며, 최종 확인 시 선택한 캐릭터와 모션이 무대에 반영되어야 합니다.

캐릭터가 하나도 없을 때의 상태도 필요합니다.
```
