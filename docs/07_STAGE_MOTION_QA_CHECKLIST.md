# 07. Stage Motion QA 체크리스트 (v1)

## 기본 동작
- [ ] 오브젝트 생성 직후 항상 `leftToRight`(좌 -> 우)로 이동한다.
- [ ] 속도는 생성 시 `0.1 ~ 0.4` 범위에서 랜덤으로 설정된다.
- [ ] 같은 오브젝트에서 이동 중 속도가 프레임마다 바뀌지 않는다.
- [ ] 드래그 종료 후에도 기존 속도가 유지된다.

## 경계/방향
- [ ] 이미지의 절반이 오른쪽 경계 밖으로 이동한 뒤 `rightToLeft`(우 -> 좌)로 전환된다.
- [ ] 이미지의 절반이 왼쪽 경계 밖으로 이동한 뒤 `leftToRight`(좌 -> 우)로 전환된다.
- [ ] 경계 판정이 캐릭터 표시 크기 기준으로 동작해 화면 밖 과이탈이 없다.
- [ ] 이동 중 캐릭터 좌우 반전이 발생하지 않는다.

## 드래그 상호작용
- [ ] 드래그 시작 시 stage motion이 즉시 정지한다.
- [ ] 드래그 중에는 자동 이동이 발생하지 않는다.
- [ ] 드래그 종료 시 현재 위치에서 드래그 시작 전 진행 방향으로 이동을 재개한다.

## 모션 합성/렌더링
- [ ] object motion과 stage motion이 동시에 적용되어도 어색하지 않다.
- [ ] 탭 반응(scale bounce) 중에도 위치/방향 반영이 자연스럽다.
- [ ] 큰 오브젝트/작은 오브젝트 모두 경계 처리에 문제가 없다.

## 신규 추가 피드백(등장 + confetti)
- [ ] 새 캐릭터 추가 시에만 입장 애니메이션(scale `0.5 -> 1.5 -> 1.0`, `800ms`)이 1회 재생된다.
- [ ] 캐릭터 드래그/리빌드/화면 회전에서는 입장 애니메이션이 재실행되지 않는다.
- [ ] 새 캐릭터 추가 순간 그림 상단보다 더 위에서 confetti가 낙하형으로 재생된다.
- [ ] 파티클이 그림 하단 근처에서 자연스럽게 사라지고 화면에 잔존하지 않는다.
- [ ] 연속 추가 시에도 프레임 저하 없이 효과가 중첩 재생된다.
- [ ] 화면 dispose/전환 시 ticker/controller 관련 예외가 발생하지 않는다.

## 제거 피드백(soft exit)
- [ ] 길게 누르면 즉시 삭제되지 않고 `removing` 상태로 전환된다.
- [ ] 제거 시작 시 stage motion/object motion/드래그가 즉시 정지한다.
- [ ] 제거 중에는 좌우 왕복/경계 반전 로직이 더 이상 적용되지 않는다.
- [ ] 제거 연출이 `1000ms` 동안 `translateY(0 -> +8 -> -36)` + `scale(1.0 -> 1.08 -> 0.58)` + `opacity(1 -> 0)`로 재생된다.
- [ ] 제거 중 탭/드래그/중복 길게 누르기 입력이 무시된다.
- [ ] 애니메이션 완료 후에만 데이터/위젯 트리에서 실제 제거된다.
- [ ] 드래그 중 길게 누르기 시 드래그가 종료되고 제거 애니메이션이 우선 적용된다.

## 회귀 확인
- [ ] 배치 수 제한(최대 10개), zIndex 정렬, 길게 눌러 제거 동작이 유지된다.
- [ ] 배경 변경 후에도 stage motion 동작이 정상 유지된다.
