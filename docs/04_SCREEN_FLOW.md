# 04. 화면 흐름 & UI 가이드

## 현재 라우트 구성

| 화면 | 경로 | 현재 역할 |
|------|------|-----------|
| 홈 | `/` | 시작 화면 |
| 내 그림 | `/library` | 저장 그림 확인/삭제 |
| 그림 가져오기 | `/capture` | 카메라/갤러리 선택 + 자동 인식 실행 |
| 이미지 크롭 | `/capture/crop` | 자동 인식 결과 기반 초기 박스 제공 + 수동 보정 |
| 미리보기 | `/capture/preview` | 저장 전 크롭 결과 확인 |
| 배경 고르기 | `/stage/background` | 5종 배경 선택 |
| 무대 | `/stage` | 캐릭터 배치 및 상호작용 |

## 실제 네비게이션 흐름

```text
[홈]
  ├─ 그림 만들기
  │   └─ [/capture]
  │      └─ 이미지 선택/촬영
  │         └─ 자동 인식 실행
  │            └─ [/capture/crop] (sourceImagePath + detectionResult 전달)
  │               ├─ 크롭 취소 -> [/capture]
  │               └─ 크롭 완료 -> [/capture/preview]
  │               ├─ 저장하기 -> 저장 처리 후 [/]
  │               ├─ 저장하고 하나 더! -> 저장 처리 후 [/capture]
  │               └─ 다시 찍기 -> [/capture]
  │
  ├─ 내 그림
  │   └─ [/library]
  │      ├─ 비어 있음 -> 그림 만들기 -> [/capture]
  │      └─ 목록 있음 -> 확인 / 삭제
  │
  └─ 놀이 시작
      ├─ 캐릭터 없음 -> 스낵바 후 [/capture]
      └─ 캐릭터 있음 -> [/stage/background] -> [/stage]
                                         ├─ + 버튼 -> 캐릭터 선택 시트
                                         ├─ 배경 버튼 -> [/stage/background]
                                         └─ 뒤로가기 -> [/]
```

## 화면별 현재 구현

### 1. 홈
- 중앙 정렬된 타이틀 `그림놀이터`
- 설명 문구 1줄
- 가로 배치 메인 액션 버튼 3개:
  - `그림 만들기`
  - `놀이 시작`
  - `내 그림`
- 라이브러리 비어 있을 때 `놀이 시작`:
  - 스낵바 표시
  - 자동으로 `/capture` 이동

### 1-1. 내 그림
- AppBar 제목: `내 그림`
- 4열 고정 그리드 카드
- 카드 내용:
  - 썸네일 이미지
  - 우상단 관리 메뉴
- 카드 탭:
  - 화면 이동 없이 `준비 중인 기능이에요` 스낵바 표시
- 빈 상태:
  - 안내 문구 2줄
  - `그림 만들기` 버튼으로 `/capture` 이동
- 삭제:
  - 관리 메뉴에서 `삭제` 선택
  - 확인 다이얼로그 후 즉시 목록 반영

### 2. 그림 가져오기
- `SafeArea` 내부 단일 컬럼
- 타이틀: `그림 가져오기`
- 필요 시 상단 피드백 배너 노출
- 큰 버튼 2개:
  - `카메라로 찍기`
  - `갤러리에서 선택`
- 이미지 선택 직후 자동 인식(`DrawingRegionDetector`) 실행
- 자동 인식 결과는 `GoRouter extra`의 `CropScreenArgs`로 CropScreen에 전달
- 처리 중 하단 `LinearProgressIndicator`

### 3. 이미지 크롭
- AppBar 제목: `이미지 크롭`
- `DetectionResult.detected=true`면 자동 인식 bbox를 초기 크롭 박스로 반영
- `DetectionResult.detected=false`면 기존 기본 크롭 상태(`viewportRect.deflate(16)`) 유지
- 화면 자체는 다음 역할을 겸함:
  - 자동 제안 박스 기반의 빠른 저장 동선 제공
  - 사용자 수동 보정(비율 선택/회전/초기화/적용)
  - 오류/취소 후 복구용 fallback

### 4. 미리보기
- AppBar 제목: `미리보기`
- 현재는 **투명 결과가 아니라 크롭 이미지를 보여줌**
- 하단 액션 3개:
  - `저장하기`
  - `저장하고 하나 더!`
  - `다시 찍기`
- 저장 중 하단 `LinearProgressIndicator`
- 저장 완료 후 스낵바:
  - 저장된 `characterId`
  - 배경 제거 품질 경고가 있으면 함께 표시

### 5. 배경 고르기
- AppBar 제목: `배경 고르기`
- 2열 그리드 카드
- 카드 내용:
  - 배경 이미지
  - 하단 반투명 라벨
- 탭 즉시 무대 이동

### 6. 무대
- AppBar 없는 전체화면 배경 무대
- 상단 오버레이:
  - 좌측 뒤로가기 아이콘
  - 우측 배경 변경 아이콘
  - 배치 수 텍스트 `n / 10`
  - 오류 카드
- 우하단:
  - `+` floating action button
  - 마지막 상호작용 후 2초 뒤 자동 fade-out
- 빈 상태:
  - 중앙 안내 문구

## 캐릭터 선택 바텀시트

### 1단계. 그림 선택
- 라이브러리 캐릭터를 4열 고정 그리드로 표시
- 카드 디자인은 `/library`와 동일한 썸네일 전용 카드 사용
- 카드 탭 시 2단계로 전환
- 빈 상태면:
  - 안내 문구
  - `그림 만들기` 버튼

### 2단계. 움직임 선택
- 좌측 또는 상단:
  - 선택한 캐릭터 미리보기
  - 현재 선택한 모션 라벨/설명
- 우측 또는 하단:
  - 모션 카드 목록
- 하단 액션:
  - 뒤로가기
  - 등장하기

## 무대 상호작용 규칙

### 탭
- 캐릭터를 앞으로 올림
- scale bounce 재생
- 재생 중 추가 탭 무시

### 드래그
- 위치 갱신
- 드래그 동안 기본 모션 정지
- 종료 후 기본 모션 재개

### 길게 누르기
- 즉시 삭제하지 않고 `removing` 상태로 전환
- 제거 시작 프레임에서 이동/드래그를 멈춘 뒤 `1000ms` 퇴장 애니메이션 재생
- 애니메이션 완료 후 실제 제거, 진행 상태는 스낵바로 표시

## 현재 문서상 주의사항
- 자동 인식은 CropScreen 진입 후 재실행하지 않고 CaptureScreen에서 1회 실행합니다.
- 자동 인식 `confidence`는 현재 UI 분기에는 사용하지 않고 디버그/실험 지표로만 유지합니다.
- 현재 무대 렌더링은 `CustomPainter`가 아니라 개별 위젯 오브젝트를 `Stack`으로 배치하는 구조입니다.
